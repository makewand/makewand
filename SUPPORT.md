# Support Policy

## Supported release line

| Version line | Status | Notes |
| --- | --- | --- |
| `v0.1.x` | Supported | Active development line |
| `< v0.1.0` | Unsupported | No backports |

## What support means

- Critical bug fixes
- Security fixes
- Release pipeline reliability fixes

Feature work is prioritized on the latest release.

## Upgrade recommendation

Always upgrade to the latest patch release in the active line.

## End of support

When a new minor line is published (for example `v0.2.x`), previous minor lines
may be marked unsupported.
